# SCDP Screenshot Capture Checklist

## Purpose
This document provides a comprehensive checklist for capturing screenshots referenced in the SCDP User Guide and Administrator Guide. Each screenshot should be saved to `docs/user-guide/screenshots/` with the filename specified.

---

## General Guidelines

**Screenshot Requirements:**
- **Resolution:** 1920x1080 minimum
- **Format:** PNG (preferred) or JPG
- **Quality:** High quality, no compression artifacts
- **Annotation:** Add arrows/highlights where needed
- **Privacy:** Remove any real client data, use sample data

**Tools Recommended:**
- **Windows:** Snipping Tool, Snip & Sketch, or ShareX
- **Mac:** Command+Shift+4 (region capture)
- **Browser:** Full page screenshots via developer tools

**Naming Convention:**
- Follow the filename exactly as listed
- Use descriptive names that match content
- Number sequentially within sections

## Capture Instructions

### For Each Screenshot:

1. **Prepare the Screen**
   - Clear any personal/client data
   - Use sample data with realistic names
   - Ensure screen is at appropriate resolution
   - Close unnecessary browser tabs/windows

2. **Capture**
   - Frame the relevant portion
   - Include enough context
   - Avoid capturing entire monitor (unless needed)
   - Ensure text is readable

3. **Annotate (if needed)**
   - Add arrows to highlight key elements
   - Add numbered callouts for step-by-step
   - Use consistent colors (red for errors, green for success, blue for info)
   - Keep annotations minimal and clear

4. **Save**
   - Use exact filename from this checklist
   - Save to `docs/user-guide/screenshots/`
   - PNG format preferred
   - Verify image quality before moving on

5. **Verify**
   - Check filename matches exactly
   - Ensure no sensitive data visible
   - Confirm image displays correctly
   - Test in markdown preview if possible

## Additional Notes

**Workflow Screenshots:**
- These can be diagrams created in tools like:
  - draw.io / diagrams.net
  - Lucidchart
  - PowerPoint
  - Miro
  - Or even hand-drawn and scanned

**Mobile Screenshots:**
- Use mobile device or browser mobile view
- Common sizes: iPhone 12 (390x844), Pixel 5 (393x851)
- Show responsive design working

**Email Screenshots:**
- Can use email preview or actual sent emails
- Ensure HTML formatting displays correctly
- Show relevant content without excessive chrome

**Diagrams:**
- Architecture diagrams can be created separately
- Keep them simple and clear
- Use consistent color scheme
- Export at high resolution
